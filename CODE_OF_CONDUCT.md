# Code of Conduct

Recall is a small open-source project maintained in spare time. Two simple
expectations apply to anyone interacting with the project — in issues, pull
requests, discussions, commit messages, or any other project channel.

## 1. Be respectful

Treat other contributors and the maintainer with basic courtesy.
The following are unwelcome and will result in the offending message being
removed and the author being warned, muted, or blocked at the maintainer's
discretion:

- Harassment, personal attacks, slurs, or discrimination of any kind.
- Hostile, condescending, or dismissive tone.
- Doxxing, threats, or any form of intimidation.
- Repeated rule-breaking after a warning.

Disagreement is fine — argue the technical merits, not the person.

## 2. Remember that Recall is free software

Recall is given away under the [Apache License 2.0](LICENSE) at no cost.
The maintainer works on it in spare time around a day job. That means:

- **Bug reports are appreciated, but not promises that anything will get
  fixed on a timeline.** Reports with clear reproduction steps and the
  affected version get priority, but timing is "when there's time."
- **Feature requests are welcome, but not commitments.** A request is an
  invitation to discuss — not an entry on a maintainer's backlog. A PR
  from a contributor is far more likely to land than a wishlist comment.
- **There are no SLAs.** No guaranteed response time for issues, pull
  requests, or even security reports. If your situation is genuinely
  urgent, fork the code and patch it — that's what the license is for.
- **Demands, pressure tactics, or entitlement are not okay.** Comments
  along the lines of "when is X going to be fixed?" with no offer to
  help, or "you need to add Y," will earn a polite redirect to this file
  and may result in the thread being closed.

In short: assume good faith, ask rather than demand, and remember that
nobody owes you their unpaid time.

## Reporting violations

If someone is violating either of the above — or if you want to report
a security vulnerability — see [SECURITY.md](SECURITY.md) for the
disclosure channel. Direct messages on third-party social platforms are
not monitored.

## Scope

This applies in all project spaces: the GitHub repository (issues, pull
requests, discussions, wikis, releases), and any other channel where
someone is identifiably acting on behalf of, or against, the project.
